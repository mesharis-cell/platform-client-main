**Supporting Documents** is a shared file area between you and the platform team, attached to the order. Use it for the kinds of files that email attachments don't handle well: floor plans, venue permits, brand guidelines, signed agreements.

<FeatureFlagNote flag="enable_attachments" />

## Where it lives

Scroll about two-thirds of the way down the order page. The card is titled **Supporting Documents**.

## What you can do

- **Download** any attachment the team uploaded for you — click the
  download button on the right of the row.
- **Read the note** under each filename, if one was left. Admins and
  logistics often add a one-line explanation so you don't have to
  open the file to know what it is.
- **Upload** your own files when the attachment-types configuration
  allows it. Types are pre-configured per platform; common ones are
  "Floor Plan", "Venue Permit", "Brand Guide", "Certificate".

## Attachment types

Every attachment has a **type label** shown above the filename. Types
are a platform-configured list — they give the logistics team a way to
quickly know what a file is without opening it. If you don't see a type
that matches what you want to upload, pick the closest one and write a
clarifying note.

<Callout kind="heads-up" title="Visibility depends on role">
Each attachment type is configured with view/upload roles. Some files
that admins or logistics upload to an order may not be visible to you;
likewise, files you upload are visible to the platform team by default
but not to other clients.
</Callout>
